# Application architecture decisions

Decision records preserve project constraints that code and tests do not fully explain. Index accepted records here.

Each record states its status, context, decision, consequences, and the observable condition that justifies reconsideration. AI may investigate options and draft a record with `Status: proposed`. A consequential application decision may use `Status: accepted` only after explicit approval from an accountable human; AI may record that approval and update the file. An application decision may strengthen local constraints. It cannot waive the installed consumer contract, suppress a `PHT` diagnostic, or permit the complete check to fail.
